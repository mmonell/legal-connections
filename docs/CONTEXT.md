# Docs Workspace

## Standards

* Guides live in `guides/`, API reference in `api/`, release notes in `changelog/CHANGELOG.md`.
* Audience per doc type:
  * `guides/` — Legal Connections team members and future developers; assume no prior context, plain language.
  * `api/` — developers integrating with or maintaining the lead API; include request/response examples.
  * `changelog/` — everyone; one entry per meaningful change, newest first.
* Keep docs in sync with code: any change to `/test` API routes or config keys updates `api/` the same day.
* All customer-facing copy quoted in docs must exist in both English and Spanish, matching the components.
